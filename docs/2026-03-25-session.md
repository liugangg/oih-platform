### 1. AF3 GPU 并发 OOM 根因修复（4项）
- `task_manager.py`: AF3 独占锁 `_af3_sem=Semaphore(1)`，防止多个 AF3 并发抢 46GB GPU

## 关键教训
1. docker exec 进程不随 API 死亡 — 必须进容器内杀进程释放 GPU
2. AF3 不能并发跑（20-40GB/task vs 46GB total）— 需独占锁
3. KNOWN_COMPLEXES chain ID 必须从实际 PDB 验证（gemmi），永远不要猜
4. Tier1 靶点用 extract_interface，Tier3 用 PeSTo，不能混用
5. B-cell epitope (DiscoTope3) ≠ PPI interface (PeSTo)
6. skills 注入必须有 token budget，bundle 要去重避免溢出
7. **禁止写临时脚本绕 pipeline.py** — 会重现已修复的 bug（chain 选择、域截断等）
