## 7. 任务取消修复（cancel_task 真正杀进程）

### 问题
前端点击终止任务后，`cancel_task()` 只修改了状态标记为 CANCELLED，但：
1. `asyncio.Task` 协程继续运行（`await coro_fn(task)` 阻塞中）
2. Docker 容器内进程继续跑，GPU 继续占用
3. 跑完后 `_run()` 把状态覆盖回 COMPLETED/FAILED

### 修复（`core/task_manager.py`）

**1. Task 新增字段**：
- `_asyncio_task`: 保存 `asyncio.Task` 引用
- `_container`: 保存容器名（从 `_TOOL_CONTAINER` 映射）

**2. `_TOOL_CONTAINER` 映射表**：tool name → Docker container name（14个工具）

**3. `submit()` 改动**：
- 存 `asyncio.Task` 引用到 `task._asyncio_task`
- 存容器名到 `task._container`
- `_run()` 内部：等待信号量后检查是否已取消；`coro_fn` 完成后不覆盖 CANCELLED 状态；catch `CancelledError`

**4. `cancel_task()` 改动**：
- 调 `task._asyncio_task.cancel()` 中断协程
- 调 `_kill_container_processes()` 杀容器内进程
- 日志记录 cancel 动作

**5. `_kill_container_processes()`**：新方法，`docker exec` 进容器内 `kill -9` 匹配进程
