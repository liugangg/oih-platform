## 今日新增功能

### 1. ADC 药物设计工具链
三个新工具，三处注册（router + qwen_tools.py + TOOL_MAP）：

| 工具 | 路由 | 队列 | 说明 |
|------|------|------|------|
| freesasa | /api/v1/design/freesasa | CPU | 抗体偶联位点预测，筛选SASA>40Å²的Lys/Cys |
| linker_select | /api/v1/adc/linker_select | 同步 | 从20条临床linker库筛选，approved优先 |
| rdkit_conjugate | /api/v1/adc/rdkit_conjugate | CPU | 7类ADC偶联反应，自动检测反应类型 |

**linker库位置**：/data/oih/oih-api/data/linker_library.json（20条，来源PMC4596905）

**7类偶联反应**：
- maleimide_thiol：马来酰亚胺+巯基（Kadcyla/Adcetris）
- nhs_amine：NHS酯+赖氨酸（VC-PABC类）
- hydrazone：腙键（pH响应释放）
- oxime：肟键（稳定可裂解）
- disulfide：二硫键（GSH裂解）
- dbco_azide：点击化学（定点偶联DAR=2）
- transglutaminase：酶促定点（返回说明）

**ADC设计完整流程**：
fetch_pdb → fpocket → rfdiffusion → proteinmpnn → af3(ipTM验证)
→ freesasa(偶联位点) → fetch_molecule(payload) → linker_select → rdkit_conjugate → chemprop(ADMET)

### 3. Bug修复

#### chemprop容器
- 根因：容器内python3 symlink → 3.10，但所有包在3.11路径
- 修复：docker exec oih-chemprop bash -c "ln -sf /usr/bin/python3.11 /usr/bin/python3"
- 容器重建会丢失此修复，需写入Dockerfile：`RUN ln -sf /usr/bin/python3.11 /usr/bin/python3`
- router修复：--accelerator cpu（避免GPU OOM）
- task_manager：chemprop/chemprop_predict加入_CPU_TOOLS

#### Qwen agent错误重试
- 问题：同一工具失败时无限重试（chemprop失败7次导致超时）
- 修复：qwen_agent.py记录连续失败次数，>=2次自动跳过

#### GET请求query params
- 问题：qwen_agent.py的executor对GET请求不传query params
- 修复：query_params = {k:v for k,v in arguments.items() if f"{{{k}}}" not in url_template}

#### 服务管理混淆
- OIH平台：/data/oih/oih-api/main.py，miniconda python，端口8080
- Gemini前端：/opt/oih-agent/app.py，oih-gemini systemd，端口8001
- 两个服务Python环境完全不同，不要混淆pip和重启命令
