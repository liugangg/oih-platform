## 5. Pipeline 代码全面核对（pipeline.py, 3355行）

### 核对结果

**✅ 已正确实现（8 项）：**
- KNOWN_COMPLEXES 5 靶标链分配全部正确，CD20 6Y4J 已移除
- Tier 分类路由（Tier1→extract_interface, Tier3→PeSTo）
- MPNN chains_to_design 自动检测最短链（gemmi），两个 pipeline 都已修复
- ADC 偶联位点只选 binder 链（`chain == _binder_ch` 过滤）
- AF3 域截取（DOMAIN_REGISTRY 7 靶标已注册）
- 热点聚类 15Å + 最多 5 个/cluster
- 评分公式 PPI 优化版（rag*0.30+pesto*0.25+conservation*0.20+sasa*0.10+electrostatics*0.15）
- DiffDock 已从 pipeline 移除

**❌ 关键缺失 → 已修复：**
- **ipSAE 未在任何 pipeline 中调用**（最大缺口）
  - 问题：AF3 验证只解析 ipTM，不调 ipSAE，无法过滤假阳性
  - 影响：CD36 DT3 路线 21 个 design ipTM>0 但 ipSAE=0.000 全是假阳性
  - 修复：在 binder_design_pipeline (L1958) 和 pocket_guided_binder_pipeline (L3143) 的 AF3 排序前添加 ipSAE 调用
  - 逻辑：ipSAE<0.01 且 ipTM 通过 → 标记 `false_positive`，`passed=False`
  - 结果字段新增 `ipsae` 和 `best_ipsae`

**⚠️ 待观察（2 项）：**
- `_extract_sequence_from_pdb` 3 处调用未传 chain_id（单链 PDB 无影响，多链复合物可能取错链。函数有 fallback 取第一条链，2026-03-27 bug fix 后不再拼接多链）
- PeSTo `gs[0][0]` 取第一条链而非指定 target 链（当前 pipeline 调用前已提取单链 PDB，实际无影响）

### 三个 Pipeline 最终版本

**drug_discovery_pipeline（6步）：**
fetch_pdb → fpocket → fetch_molecule → GNINA docking → AF3 complex → GROMACS 10ns MD

**binder_design_pipeline（7步+ipSAE）：**
fetch_pdb → Tier1 auto-hotspot → RFdiffusion → ProteinMPNN(auto chain) → AF3(域截取) → **ipSAE验证** → FreeSASA+Linker+RDKit ADC

**pocket_guided_binder_pipeline（16步+ipSAE）：**
fetch_pdb → RAG两层搜索 → fpocket+P2Rank+DT3(并行) → FreeSASA → conservation+electrostatics → PPI评分(5D)+Qwen选择 → RFdiffusion+BindCraft(并行) → ProteinMPNN(auto chain) → ESM2 → AF3(域截取) → **ipSAE验证** → FreeSASA偶联 → Linker → MMAE → RDKit(DAR=4)

## 7. Thinking Mode 关闭

### 问题
Qwen3 的 thinking mode 生成大量思考内容（500-2000 tokens），每轮推理需要 60-180 秒，极慢。

### 修复
- `enable_thinking: False` 应用于 `chat()` 和 `chat_stream()` 两个方法
- thinking_budget 设为 0
- 简单消息用 100 字短 prompt（不用 9000 字完整 prompt）

### 效果
简单消息 ~15 秒（之前 30+ 秒），工具调用轮次从 60-180 秒降到 30-60 秒。

## 8. 快速路由系统（最大优化）

### 问题
固定流程的任务（如"对接 CD36 和棕榈酸"）不需要 Qwen 推理，但每次都要等 30-60 秒 Qwen 选工具。14B 模型还经常选错（fetch_molecule 查蛋白质、UniProt ID 当序列传）。

### 实现
`detect_fast_route()` 静态方法：正则匹配用户消息 → 直接构造工具调用序列 → 跳过 Qwen。

### 11 条路由

| # | 触发词 | 路由 | 自动执行 |
|---|--------|------|---------|
| 1 | alphafold/af3/结构预测 | af3_predict | fetch_pdb → 提取序列 → alphafold3_predict |
| 2 | 对接/dock | docking | fetch_pdb + fetch_molecule → dock_ligand(gnina) |
| 3 | 设计binder/抗体/靶向 | binder_pipeline | pocket_guided_binder_pipeline |
| 4 | admet/毒性/溶解度 | admet | fetch_molecule → chemprop_predict |
| 5 | 口袋/pocket/结合位点 | pocket | fetch_pdb → fpocket |
| 6 | 药物发现/全流程 | drug_discovery | drug_discovery_pipeline |
| 7 | 分子动力学/gromacs/模拟 | md_simulation | fetch_pdb → gromacs(10ns) |
| 8 | 表位/epitope/B细胞 | epitope | fetch_pdb → discotope3 |
| 9 | 下载PDB | fetch_pdb | fetch_pdb |
| 10 | 获取分子/SMILES | fetch_molecule | fetch_molecule |
| 11 | 文献/论文/pubmed | literature | search_literature |

### 辅助功能
- 靶标 PDB 映射表（17 个已知靶标 → PDB ID）
- 中文药名翻译表（12 种常见药物 → 英文名，PubChem 不支持中文）
- PDB ID 正则兼容中文上下文（`(?<![A-Za-z0-9])` 替代 `\b`）
- AF3 路由自动从 PDB 文件提取序列（fetch_pdb 不返回 sequence 字段）

### 测试结果（test_fast_routes.py，8/8 通过）
| 路由 | 耗时 | 状态 |
|------|------|------|
| PDB下载 | 6.6s | ✅ |
| 小分子获取 | 2.5s | ✅ |

| 口袋检测 | 2.1s | ✅ |
| ADMET预测 | 2.9s | ✅ |
| 表位预测 | 1.4s | ✅ |
