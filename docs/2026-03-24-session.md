### 13. AF3 GPU 并发 OOM 修复（3项）
- **根因**: `docker exec` 进程 parent 是 containerd-shim，kill API 杀不掉容器内进程 → GPU 显存泄漏
- **修复1**: `task_manager.py` — AF3 独占锁 `_af3_sem=Semaphore(1)`，防止多个 AF3 并发抢 46GB GPU
- **修复2**: `structure_prediction.py` — AF3 超时 1200s→3600s/5400s/7200s（MSA jackhmmer 需 30min+）
- **修复3**: `docker_client.py` — 新增 `_kill_container_processes()`，超时时进容器内杀进程释放 GPU
- **修复4**: `pipeline.py` — timeout 加入致命错误列表，不再无限重试

## 关键教训
1. B-cell epitope ≠ PPI interface (CD36 proof: ipSAE=0.000 vs 0.193)
2. PeSTo 对复合物 PDB 必须提取单链 (PD-L1: 0.44→0.994)
3. 更多 designs 不一定更好 (n=50 ipTM=0.18 vs n=10 ipTM=0.55)
4. Pipeline 必须在 CPU 队列，否则 GPU deadlock
5. ipSAE > ipTM 作为结合质量指标
6. docker exec 进程不随 API 死亡 — 必须进容器内杀进程释放 GPU
7. AF3 不能并发跑（20-40GB/task vs 46GB total）— 需独占锁
